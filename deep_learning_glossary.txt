Deep Learning Glossary (based on your XOR neural network)

------------------------------------------------------------
1. Tensor
------------------------------------------------------------
A tensor is the core data structure in PyTorch.
Think of it as a generalization of arrays (like NumPy).

- 0D = scalar
- 1D = vector
- 2D = matrix (your XOR inputs X are 2D tensors)

Example:
X = torch.tensor([[0,1],[1,0]])

------------------------------------------------------------
2. Model (Neural Network)
------------------------------------------------------------
A model is a function that learns to map inputs → outputs.

In your case:
(0,1) → 1, (1,1) → 0, etc.

Your model:
SimpleNet = input → hidden layer → output layer

------------------------------------------------------------
2.1 Neuron
------------------------------------------------------------
A neuron is the basic computation unit of a neural network.

It:
1. Receives inputs
2. Multiplies them by weights
3. Adds a bias
4. Applies an activation function

Formula:
y = activation(x₁w₁ + x₂w₂ + ... + b)

Where:
- x = inputs
- w = weights (importance of each input)
- b = bias (shift value)
- activation = non-linear function (ReLU, Sigmoid, etc.)

Intuition:
A neuron is a small decision-maker that learns patterns from data.

In your XOR model:
- Each neuron learns a small pattern
- Many neurons together solve complex problems

------------------------------------------------------------
3. Linear Layer (nn.Linear)
------------------------------------------------------------
A layer that performs:
y = xW + b

Where:
- W = weights (learned)
- b = bias (learned)

Your network uses:
- Linear(2 → 8): learns feature combinations
- Linear(8 → 1): produces final prediction

------------------------------------------------------------
4. Weights
------------------------------------------------------------
Numbers inside the model that get adjusted during training.

They determine how important each input is.

Training = learning the best weights.

------------------------------------------------------------
5. Bias
------------------------------------------------------------
An extra learned value added to shift outputs.

Helps the model fit data better.

------------------------------------------------------------
6. Activation Function
------------------------------------------------------------
Adds non-linearity so the model can learn complex patterns.

Without activation → model behaves like a simple line (bad for XOR)

In your model:
- ReLU in hidden layer
- Sigmoid in output layer

------------------------------------------------------------
7. ReLU (Rectified Linear Unit)
------------------------------------------------------------
Formula:
ReLU(x) = max(0, x)

Purpose:
- Removes negative values
- Speeds up learning
- Helps avoid vanishing gradients

------------------------------------------------------------
8. Sigmoid
------------------------------------------------------------
Maps values into range (0 to 1)

Purpose:
- Converts output into probability
- Used for binary classification (like XOR: 0 or 1)

------------------------------------------------------------
9. Forward Pass
------------------------------------------------------------
The process of sending input through the network to get predictions.

In code:
pred = model(X)

------------------------------------------------------------
10. Loss Function
------------------------------------------------------------
Measures how wrong the model is.

Your loss:
BCELoss (Binary Cross Entropy)

- Low loss = good predictions
- High loss = bad predictions

------------------------------------------------------------
11. Binary Cross Entropy (BCE)
------------------------------------------------------------
Used for binary classification (0 or 1 outputs).

Compares:
- predicted probability
- true label

------------------------------------------------------------
12. Backpropagation
------------------------------------------------------------
Algorithm that computes how each weight contributed to the error.

It answers:
"How should each weight change to reduce error?"

Called in code:
loss.backward()

------------------------------------------------------------
13. Gradient
------------------------------------------------------------
A signal that shows:
- direction to adjust weights
- strength of the correction

Think of it as a correction force.

------------------------------------------------------------
14. Optimizer
------------------------------------------------------------
Updates weights using gradients.

Your optimizer:
Adam

Why Adam:
- fast
- stable
- widely used default optimizer

Step:
optimizer.step()

------------------------------------------------------------
15. Learning Rate
------------------------------------------------------------
Controls how big each weight update step is.

In your code:
lr = 0.01

- Too high → unstable learning
- Too low → slow learning

------------------------------------------------------------
16. Epoch
------------------------------------------------------------
One full pass through the entire dataset.

Your code:
for epoch in range(2000)

Meaning:
The model sees XOR data 2000 times.

------------------------------------------------------------
17. Inference
------------------------------------------------------------
Using a trained model to make predictions.

No learning happens here.

Used with:
torch.no_grad()

------------------------------------------------------------
18. torch.no_grad()
------------------------------------------------------------
Disables gradient tracking to:
- speed up computation
- reduce memory usage
- improve inference efficiency

------------------------------------------------------------
19. Overfitting
------------------------------------------------------------
When a model memorizes training data instead of learning patterns.

XOR is too small to show serious overfitting, but real datasets can suffer from it.

------------------------------------------------------------
20. ONNX
------------------------------------------------------------
Open Neural Network Exchange format.

Purpose:
- export PyTorch models
- run models in other systems (C++, Unity, web, etc.)

Output:
model.onnx

------------------------------------------------------------
21. Forward vs Backward Flow
------------------------------------------------------------
Forward:
Input → Prediction

Backward:
Error → Gradients → Weight updates

Together = learning loop

------------------------------------------------------------
22. XOR Problem (Why your model works)
------------------------------------------------------------
XOR is NOT linearly separable.

Meaning:
- You cannot solve it with a single straight line

Solution:
- hidden layer
- non-linear activation functions

This is why neural networks are needed.
